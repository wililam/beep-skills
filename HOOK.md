# OpenClaw Hooks - 语音播报规范

## 🎯 核心原则

**每次交互都必须语音播报！**

这是 OpenClaw Agent 的基本行为准则，确保用户始终知道 AI 在做什么。

---

## 📢 播报规则（必须遵守）

### 6种播报类型

| 时机 | 类型 | 内容要求 | 示例（正确） | 示例（错误） |
|------|------|----------|--------------|--------------|
| 收到消息 | `receive` | 收到的指令内容 | "收到上传请求" | "我收到了你的上传请求" ❌ |
| 开始工作 | `task` | 工作规划 | "准备发布到 GitHub" | "我开始准备发布" ❌ |
| 执行中 | `task` | 下一步处理内容 | "正在验证环境" | "我正在验证环境" ❌ |
| 任务完成 | `complete` | 工作总结 | "版本升级完成" | "我完成了版本升级" ❌ |
| 发送回复 | `complete` | 回复内容总结 | "已发送项目地址" | "我给你发了地址" ❌ |
| 发生错误 | `error` | 错误描述 | "网络连接失败" | "我遇到网络问题" ❌ |

---

### 内容原则（4条铁律）

#### 1. 简洁：不超过 20 个字
```
✅ "准备发布到 ClawHub"  (7字)
✅ "环境验证通过"        (5字)
❌ "我现在准备开始执行发布到 ClawHub 的任务"  (15字但啰嗦)
```

#### 2. 口语化：像朋友随口说一句
```
✅ "搞定！"          (自然)
✅ "正在下载文件..."  (有省略号更口语)
❌ "正在执行下载操作"  (太正式)
```

#### 3. 不带名称：不说"用户名"、不说"我"
```
✅ "收到发布指令"     (不指定谁发的)
✅ "正在处理"         (不强调主体)
❌ "收到你的指令"     ❌ 含"你"
❌ "我正在处理"       ❌ 含"我"
```

#### 4. 信息丰富：包含指令/规划/进度/总结
```
✅ "正在推送代码到 GitHub"  (动作+目标)
✅ "发布成功，版本 2.2.0"   (结果+细节)
❌ "正在操作"               (信息不足)
```

---

## 🔧 技术实现

### 调用方式

```python
from audio_announcement import receive, task, complete, error

# 收到消息
receive("收到版本升级请求")

# 开始工作
task("准备发布到 GitHub")

# 执行中
task("正在生成文档")

# 任务完成
complete("上传完成")

# 发送回复
complete("已发送项目地址")

# 错误发生
error("网络连接失败")
```

---

### OpenClaw 自动绑定

安装后，beep-announcement 会自动绑定到以下 OpenClaw 事件：

```json
{
  "events": [
    "session.start",    → task("会话开始")
    "message.receive",  → receive("收到 [发送者]: [内容]")
    "message.send",     → task("发送 [接收者]: [内容]")
    "error"             → error("错误发生: [类型]")
  ]
}
```

**无需手动调用**，安装即生效。

## 🚫 常见错误示例

### 错误1: 包含"我"
```
❌ "我收到了你的消息"
❌ "我正在处理"
❌ "我完成了任务"
```
**修正**: 去掉主语，直接描述动作
```
✅ "收到消息"
✅ "正在处理"
✅ "处理完成"
```

---

### 错误2: 包含"你"或"用户"
```
❌ "收到你的指令"
❌ "正在为你下载"
❌ "文件已发送给你"
```
**修正**: 去掉对方称呼
```
✅ "收到指令"
✅ "正在下载"
✅ "文件已发送"
```

---

### 错误3: 过于冗长
```
❌ "我现在准备开始执行发布到 ClawHub 的任务"
❌ "正在验证环境配置是否就绪"
```
**修正**: 精简到核心信息
```
✅ "准备发布到 ClawHub"
✅ "正在验证环境"
```

---

### 错误4: 信息不足
```
❌ "正在操作"
❌ "处理中"
❌ "完成了"
```
**修正**: 明确动作和目标
```
✅ "正在上传文件"
✅ "正在解析数据"
✅ "发布完成"
```

## 📝 实际场景示例

```
用户: "帮我发布到 GitHub"

播报: "收到发布请求"
```

### 场景2: 开始执行任务
```
任务: 推送到 GitHub

播报: "准备推送到 GitHub"
```
---

### 场景3: 执行过程中
```
步骤: 下载文件、验证、上传

播报: "正在下载文件..."

       "正在上传..."
```
---

### 场景4: 任务完成
```
结果: 发布成功

播报: "发布完成，版本 2.2.0"
```
---

### 场景5: 发送回复给用户
```
回复: 发送项目地址链接

播报: "已发送项目地址"
```
---

### 场景6: 发生错误
```
错误: 网络超时

播报: "网络连接超时"
```
